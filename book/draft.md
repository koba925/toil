#### 式と文

Toilでは、すべてが「式」です。

「プログラム」＝「式」

何かを評価すれば必ず値を持ちますし、関数ではreturnと書かなくても最後に評価した式の値が返されます。

「式」じゃないものって何かっていうと「文」です。
「式」と「文」の違いは何かっていうとざっくりこんなところです。
（副作用が・・・といった話もあるけれどもここでは触れない）

* 「式」は値を持つ。「文」は値を持たない。
* （文のある言語では）「式」は「文」のパーツになっている。

Pythonで言えば、`if a == b: print(1) else: print(2)`は文です。
`a == b`や`print(1)`、`print(2)`が式で、これらは文のパーツになっていて値を持ちます。
`a == b`はもちろん`True`や`False`の値を持ちますし、あまり意識してないと思いますが`print()`関数は`None`という値を持ちます。
それに対し、`if a == b: print(1) else: print(2)`全体は値を持ちません。
`1 if a == b else 2`は式で、`1`や`2`という値を持ちます。

`+`・`==`・`print`の式は以下のように書くことにします。

タプルの最初の要素に関数名、2番目の要素には引数の配列（list）、という形です。

[a, b]のようなものをPythonではリストと呼びますが

本格的に言語処理系を実装する場合は、クラスを作ることが多いのですが、
Toilではシンプルに書けてコンパクトに表示できるタプルを使います。
コードを書くにも、デバッグなどでprintなどするにも、タプルの方が短くて済むので学習には便利かなと。

`self.eval(a) + self.eval(b)`のところで警告が出てると思います。
`self.eval(a)`の結果と`self.eval(b)`の結果が足し算できるとは限らないためです。

ちゃんとした言語を作るときは、足し算できるかどうか確認し、できなければエラーにしてくださいね。

#### タプルと配列

タプルと配列（list）を使いました。
どちらもいくつかの値をまとめるために使うものです。
ではなぜ両方使っているのでしょうか？

配列は同類のものを並べたもの。個数は決まっていない。
タプルは異質のものを並べたもの、個数は決まっている。

こんな感じで使い分けています。

Pythonでは、配列はミュータブルでタプルはイミュータブルという違いもあります。
Toilでもたまにその違いが顔を出します。


matchではリストとタプルが区別されません。びっくりです。

```py
match [2, 3]:
    case (a, b): print("tuple")
    case [a, b]: print("list")
```

とやると`tuple`と表示されてしまいます。
これは`case (a, b):`や`case [a, b]:`はシーケンスパターンと呼ばれ、
タプルも配列もどちらも「シーケンス」としていっしょくたにされます。

本来 `case ("seq", exprs) if isinstance(expr, list)`などと書いてタプルと
配列を区別するのがよいプラクティスです。
Toilでは、のちほど「もし配列だったら」という確認が入り、それ以外はすべてタプルと
判断できるようになるためひとつひとつ確認はしていません。
厳密には dequeやnamedtupleもマッチしてしまうのでダメなんですが
Toilではタプルか配列しか扱わないのでいいことにしてます。


タプルについてもうひとつ。

タプルを`(2, 3)`のように書くと、カッコがタプルを表す記号のように思うかもしれませんが
タプルの本体はカンマなのです。
カッコは書かなくてもタプルです。

```
>>> (2, 3)
(2, 3)
>>> 2, 3
(2, 3)
```

Pythonには`a, b = b, a`と書くと`a`と`b`を入れ替えられるよ！という技がありますが
`(a, b) = (b, a)` と同じ意味なんですね。

とてもおもしろい仕様なんですがじゃあ `print(2, 3, 4)` の `2, 3, 4` は
タプルなのかというとそうではありません。
ちょっと気持ち悪いところでもあります。

#### 条件分岐 0104_if

もうひとつ制御構造を作ってみましょう。
こんどは「選択」(Selection)つまり`if`です。

`if`はこんな形にします。
Pythonの`if`では`else`がなかったり`elif`で続いたりしますが`eval()`ではこの形のみ扱います。

`("if", [条件式, 帰結式, 代替式])`

帰結式 (Consequent) は条件式が真の時に評価される式
代替式 (Alternative) は条件式が偽のときに評価される式
帰結式、代替式と呼ぶとなじみがない感じだと思いますのでThen式、Else式と呼んでいきます。

ポイントとなるのは条件式が真の時は帰結式は評価されず、
条件式が偽の時はThen式は評価されない、ということです。
当たり前に思えるかもしれませんが大事なところです。
あとでもう一度話します。


`if`ではかならず

プログラム ＝ 式
式 ＝ 定数 または 順次 または 選択
定数 ＝ None または bool または int
順次 ＝ ("seq", [式, 式, ...])
選択 ＝ ("if", [条件式, 帰結式, 代替式])
